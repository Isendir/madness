Kanban: Status report 2012-07-23 20:40 Technology Kanban

I started working on the personal kanban application I wrote about earlier, there have been a couple of changes regarding the implementation, and a few mockups of the user interface are ready to be showcased.

One of the major shifts in design is that the application will not be based on a git-backed storage. Instead, it will be built upon a MongoDB, and tools will be provided to export, import and merge databases, making disconnected work possible. The way I imagine that, is that the application will have a replay-able log of what happend, so merging will be as easy as replaying the log. If conflicts arise, there'll be an API to ease resolving them.

The goal is to have something that's easy to deploy either locally, or on, say, heroku, that's one of the reasons I dropped the idea of using git as the storage backend. The other is that I found it unreasonably complicated to work with git from Clojure. On the other hand, Monger has been a great pleasure to work with so far.

I have not written a single line of code yet, figuring I will design the user interface first, at least roughly, so I will know what APIs I will need to make it work. The results are below, but keep in mind that these are a work in progress, and I'm not a designer, nor an UX expert or anything.
