Madness! 2012-06-27 23:50 News Technology Clojure

This site has a long history. It started out as a database-driven, dynamic thing, based on Django + Zinnia, then it turned into an Octopress-powered statically generated site.

Today, I switch again. Today, I praise the power of Enlive, Bootstrap, and Clojure, for they made it possible for me to move on.

There always were annoying little things that bothered me about Octopress: from the language, through the complex and fragile update mechanism, to the silly source layout. Whenever I tried to update, or pull a plugin or extension in, I had to fight with the system. I don't like fighting the system.

I like simple things. Simple things, that allow me to control them as I see fit, and don't try to shoehorn me into doing things their way.

Why would I have to copy the theme?

One part I did not understand about Octopress, is why the complex theme & style update mechanism? It's in version control already! How about I create a branch, change the theme as I see fit, and then merge if I want to update? Would that be so hard? No. Granted, that only supports a single theme within a branch, but I can live with that.

Why do I have to learn a new templating language?

There's a very wide-spread markup language out there, it is called HTML. When I want a new theme, I want to write HTML. I do not want any fancy extra moustaches there, nor any other non-sense. I want to be able to preview the theme just by loading a single file into my browser (of course, with the CSS and everything too).

What did I do about it?

I ended up ditching Octopress (yay, no more Ruby on my systems!), and wrote my own static site generator. I looked around at first, to see if I can find one that matches my needs:

  • Written in a sane language (Perl, Python, Clojure)
  • Simple: the theme is simple HTML/CSS (Less & SASS are ok), and nothing else
  • It supports usable archives and feeds: whole site and per-tag alike
  • Simple configuration: whatever can be in the template, should be in the template. I like my configs minimal.

Not many requirements, right? But none of the static site generators I looked at were able to fulfill all these needs. So, I went ahead and wrote my own, that does all of the above.

While doing so, I learned to love Enlive, because it's insanely powerful. This whole site here is made up of two template files: one for the HTML, another for the Atom feeds. And that's it. With Enlive, I pull in snippets from the template, fill in placeholders, clone & repeat parts of it, and transform it into this site. It is actually quite simple, the whole code is about 600 lines of code, but once I'm happy with the feature set, I will do a refactor round and trim it down a little, most probably.

I learned to love Bootstrap, because it made it so much easier to design the look, and have a flexible, responsive layout, that - as far as I could test - works much better on smartphones than my old site did!

I already loved Clojure, but now I love it even more. It was a very interesting, and challenging experiment to build a static site generator in a functional language. But in the end, composing parts turned out to be a great tool.

Once cleaned up a little, the code of Madness and this theme will all be available at a neighborhood git repository. For now, it's a bit of a mess, because I was still learning my way around. You do not want to see the code in its current form. Though, those interested, will be able to, as it's part of the git history.

Soon! Madness will consume you. *cackles madly*
